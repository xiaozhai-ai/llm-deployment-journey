### 场景 3：错误处理测试

#### 3.1 不支持的文件格式

**测试**：上传 `test.xlsx` 或其他非 PDF/DOCX/TXT 文件

**预期**：
- ❌ 显示错误：📄 不支持的文件格式，请上传 PDF/Word/TXT 文件
- ✅ 不崩溃，界面正常

#### 3.2 错误的 API 密钥

**测试**：修改 `start.bat` 中的 `LLM_API_KEY` 为错误值

**预期**：
- ❌ LLM 分析失败
- ✅ 降级到规则分析
- ✅ 显示：🔑 API 密钥无效，请检查 LLM_API_KEY 配置

#### 3.3 网络超时

**测试**：断开网络或设置错误的 `LLM_API_BASE`

**预期**：
- ❌ LLM 请求超时
- ✅ 自动重试 3 次
- ✅ 降级到规则分析
- ✅ 显示：⏱️ AI 分析超时，已使用规则分析结果

## 🐛 常见问题

### Q1: 启动报错 "No module named 'gradio'"
**解决**：`pip install -r requirements.txt`

### Q2: ChromaDB 安装失败
**解决**：可以暂时不安装，系统会降级到关键词模式

### Q3: LLM 调用超时
**解决**：
- 检查网络连接
- 检查 API 密钥是否正确
- 检查 Base URL 是否可达

### Q4: 审查结果风险太少
**解决**：
- 勾选"启用 AI 深度分析"
- 检查 `config/legal_rules.yaml` 规则是否完整

### Q5: 修订建议为空
**解决**：
- 确保 LLM API 可用
- 检查是否有可操作的风险项
