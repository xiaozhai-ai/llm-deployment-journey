# Changelog

本文件记录法务审查 Agent 的版本演进历史。

## [2.5.0] - 2026-05-31

### ✨ 新增
- **Tool-Calling Agent**: ReAct 模式推理循环，AI 自主调用法规/判例检索工具
  - 4 维停滞检测（重复调用/空结果/信息增益/推理收敛）
  - 自我反思机制：结论输出前交叉核对工具结果与引用准确性
  - 流式输出支持，前端实时展示推理过程
- **溯源高亮**: 风险项 ↔ 原文条款双向跳转，展示条款位置、引用法条、置信度
- **人工修正反馈闭环**: 用户标记误报 → 反馈注入 LLM Few-shot → 持续优化
- **知识库新鲜度检测**: 法条/判例状态标注、过期预警、自动提示更新
- **多路精准召回**: 向量语义 + 关键词扩展 + 法律术语强制召回，三路融合排序
- **法律术语词典**: 解决"定金"≠"订金"等术语精确匹配问题
- **DOC（WPS）格式支持**: 通过 olefile 解析 OLE 容器中的 WordDocument 流

### 🔧 改进
- Token 优化：单次审查 token 消耗降低约 55-60%（25000 → 10000-12000）
- 超长文档自动分段（首/中/末 3 段），控制 token 消耗
- 历史消息智能压缩，严格保持 tool-calling 消息配对完整性
- 修订建议封顶 5 条，避免 token 浪费
- Gradio Chatbot 升级为 `type="messages"` 格式

### 🐛 修复
- SSE 流式解码中文乱码：`requests` 库默认 ISO-8859-1 → 显式设置 UTF-8
- API 端点配置：`/anthropic` → `/v1`（兼容 OpenAI 格式）
- 启动脚本硬编码 API Key → 改为 `.env` 环境变量
- `_temp_files` 线程安全问题 → 添加 `threading.Lock`
- `validate_api_key()` 同步阻塞 → 改为异步
